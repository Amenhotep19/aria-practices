Accessibility Features

  1. A switch should be used when the change is automatically saved. There should be no need to activate a “save” button for the change to take affect.
  2. There is a fallback role of checkbox in case the used assistive technology doesn’t support the switch role.

Keyboard Support

Key Function
Tab
  • Moves keyboard focus to the switch.
Space
  • Toggle switch between checked and unchecked.

Role, Property, State, and Tabindex Attributes

Role Attribute Element Usage
switch div
  • The role="switch" attribute identifies the div element as an ARIA switch.
  • The switch widget needs a tabindex="0" value.
tabindex="0" div
  • The div["switch"] is identified as an ineractive element and is added to the tab order of the page by setting the tabindex="0".
aria-checked="false" div
  • Indentifies the switch button as unchecked.
  • CSS attribute selectors (e.g. [aria-checked="false"]) are used to synchronize the visual states with the value of the aria-checked attribute.
aria-checked="true" div
  • Indentifies the switch as checked.
  • CSS attribute selectors (e.g. [aria-checked="true"]) are used to synchronize the visual states with the value of the aria-checked attribute.
